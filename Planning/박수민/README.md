### 250421 

- 기획

    ```
    **PopZone**은 사람들이 특정 장소에 몰렸을 때,  
    해당 공간을 중심으로 **실시간 소통·물건 나눔·정보 교류가 가능한 임시 커뮤니티 공간을 자동 생성**해주는 위치 기반 소셜 플랫폼입니다.

    * * *

    🧩 핵심 문제의식
    ----------

    *   도시 속 ‘지금, 여기’의 정보/교류 수요는 많지만  
        SNS나 커뮤니티는 **시간/공간 지연**이 크고 즉시성이 부족함
        
    *   공연장, 야구장, 마켓, 행사장 등에서 생기는 **실시간 나눔·도움 요청·소통**의 필요를 앱으로 해결하고자 함
        

    * * *

    🌟 주요 기능
    --------

    1.  **PopZone 자동 생성**
        
        *   GPS 기반으로 일정 반경 내 사용자가 일정 수 이상 모이면 해당 장소에 PopZone 생성
            
        *   장소 유형에 따라 반경이 유동적으로 설정됨 (예: 카페 50m / 야구장 200m)
            
    2.  **실시간 소통 (익명 채팅)**
        
        *   즉석 채팅, 제한 시간 60분
            
        *   줄 정보, 물건 잃어버림, 주변 소음 공유 등 목적별 말풍선 분리
            
    3.  **물건 나눔·교환 기능**
        
        *   “마스크 있어요”, “충전기 빌려드려요”, “에어팟 찾습니다”
            
        *   위치 기반 나눔 리스트 매칭
            
    4.  **후기/신뢰도 시스템**
        
        *   교류가 완료되면 “👍 도움이 됐어요” 등 평가 가능
            
        *   장소별 신뢰도 누적 → 안전한 공간 문화 형성
            

    * * *

    🛠️ 장소 유형별 반경 설정
    ----------------

    | 장소 유형 | 기본 반경 | 이유 |
    | --- | --- | --- |
    | 카페/도서관 | 30~50m | 소규모 공간, 밀접 위치 |
    | 공연장/전시장 | 100~200m | 넓은 공간, 입구/출구 간 거리 큼 |
    | 야구장/체육관 | 150~300m | 관중석/입장구역/주차장 등 다양 |

    → 장소 유형은 **고정 POI DB + 사용자 선택 + 실시간 밀도 분석**으로 판단

    * * *

    🧠 기술 구조 요약
    -----------

    | 영역 | 기술 |
    | --- | --- |
    | 실시간 위치 감지 | GPS + H3.js 공간 클러스터링 |
    | PopZone 생성 로직 | 사용자 수 + 장소 유형 기반 자동화 |
    | 소통 | Firebase Realtime DB / WebSocket |
    | 교환 매칭 | MongoDB 기반 물품 DB + 반경 필터링 |
    | 반응형 UI | React Native + TailwindCSS |
    | 인증 | 익명 로그인 + 세션 기반 활동 관리 |

    * * *

    🎯 차별화 포인트
    ----------

    *   고정된 커뮤니티가 아닌, **장소에 따라 순간적으로 생성/사라지는 커뮤니티**
        
    *   반경과 기능이 **장소의 특성과 사람의 밀도에 따라 자동으로 맞춰짐**
        
    *   실질적인 물건 공유/정보 요청/실시간 대화가 가능
        
    *   공공성과 상업성을 모두 지닐 수 있음 (행사/재난/축제/상점 홍보 등)
        

    * * *

    📍 활용 예시
    --------

    *   공연장 앞: 응원봉 교환, 잃어버린 물건 찾기
        
    *   야구장 입장 대기: 줄 위치 공유, 응원용품 나눔
        
    *   플리마켓: 즉석 교환, 재고 공유
        
    *   대학 캠퍼스: 시험 정보, 팀플 모집, 간식 나눔
        
    *   재난 대피소: 생필품 공유, 구조 요청, 안전 정보
    .
    ```

    ```🍊 프로젝트 개요 – 오렌지 당도 예측 앱
    1. 🎯 프로젝트 명
    오렌지오렌지지 – 스마트폰으로 찍는 오렌지 당도 예측기

    2. 🧩 기획 배경
    오렌지와 같은 과일은 외관으로 익은 정도를 가늠하기 어렵고, 맛도 상품마다 큰 편차가 있음.
    소비자는 "어떤 게 더 달까요?"를 판단할 수 없어 주관적으로 구매하게 되며, 이는 소비 만족도 저하로 이어짐.
    특히 온라인이나 간접 구매 환경에서는 당도 확인이 불가능함.

    ➡️ 스마트폰 카메라만으로 과일의 당도를 예측해주는 보조 도구가 필요함

    3. 🧠 서비스 개요
    사용자가 오렌지를 스마트폰 카메라로 촬영하면,
    과일 껍질의 색상(RGB)과 표면 특성을 분석하여,
    당도(Brix 값) 등급을 예측하고 구매 판단을 도와주는 모바일 서비스

    4. 📱 주요 기능

    기능	설명
    📷 오렌지 촬영	앱 내 카메라 또는 사진 업로드 기능
    🎨 색상 추출	촬영 이미지에서 R/G/B 및 밝기값 추출
    📊 당도 예측	머신러닝 모델로 예측한 당도 등급 제시 (예: High, Medium)
    📚 기록 관리	당도 예측 결과 저장 + 비교 가능
    🔍 시세/정보	오렌지 품종별 평균 당도 정보 또는 가격 비교 
    ```
